# PPTX 媒体优化路线图

## 目标

在不破坏教学内容、不覆盖原始文件的前提下，降低 PPTX 体积，并为照片类、实验实物类低清素材预留可选清晰化能力。

## 分阶段实施

### 第 1 阶段：低风险媒体扫描与保守压缩

- 扫描 `ppt/media/*`，统计格式、尺寸、体积、最大媒体文件。
- 只对 JPEG/JPG 做保守重编码。
- 只在新文件更小且节省比例达到阈值时替换。
- PNG、GIF、SVG、EMF、WMF、MP4 默认只统计，不自动改。
- 输出 `.media-optimized.pptx`，不覆盖 `.normalized.pptx`。

### 第 2 阶段：接入开源压缩工具

- 已落地可选路径：`Optimize-PptxMedia.ps1 -UseSharp` 可调用开源 `sharp/libvips`。
- 已落地便携工具：`tools/vendor/oxipng-10.1.1/`。
- PNG 无损优先：`oxipng`，未找到时回退 `sharp/libvips`。
- PNG 有损可选：`pngquant`，默认关闭。
- JPEG 高质量压缩：`sharp/libvips`、`mozjpeg` 或 `jpegtran`。
- 通用处理：`ImageMagick` / `libvips` / `sharp`。

### 第 3 阶段：选择性清晰化

- 只对照片、实验实物图、低清自然图启用。
- 已落地只读候选扫描：`Export-PptxImageCandidates.ps1`，先输出候选清单、抽检图片和缩略总览，不直接改 PPTX。
- 已落地便携工具：`tools/vendor/realesrgan-ncnn-vulkan-20220424/`。
- 已落地清晰化抽样验证：`Invoke-PptxImageEnhancementProbe.ps1`，只生成前后对比，不写回 PPTX。
- 抽样验证会保留 4 倍增强 PNG，同时默认生成原尺寸 JPEG 85 交付候选，用于评估“清晰化后再压缩”的可行性；确需更多像素时再切到 2 倍交付。
- 已落地受控写回：`Apply-PptxImageEnhancement.ps1`，只替换抽检成功、JPEG 对 JPEG、尺寸相同、体积不增的媒体。
- 可选工具：Real-ESRGAN / Upscayl / `realesrgan-ncnn-vulkan`。
- 默认不处理公式、光路图、表格、截图、文字密集图片。
- 本机实测 `realesrgan-x4plus -s 2` 会出现分块错位伪影；抽样验证默认使用 `-s 4`，写回前仍需体积和画面双重确认。
- 清晰化后必须重新导出 PDF/PNG，并生成前后对比。

### 第 4 阶段：课堂交付稳定性

- 批量导出遇到 `RPC_E_CALL_REJECTED` 时，自动重启 PowerPoint 并对单文件重试。
- 扫描 `TargetMode="External"` 外部关系，输出视频、外部图片、网页链接依赖清单。
- 外部图片或本地视频链接缺失时必须在报告中暴露，不默认删除链接。

## 当前任务清单

- 已将媒体优化脚本纳入 `tools/Test-ToolkitFiles.ps1` 自检。
- 使用 final 目录中的规范化 PPTX 实跑媒体优化。
- 对优化后的 PPTX 重新导出 PDF 和图片，检查是否可打开、可导出、版面无明显异常。
- 记录节省比例、替换数量、失败数量和仍需人工判断的图片类型。
- 对 final 目录运行图片候选扫描，确认哪些图片适合进入清晰化抽检。
- 对低清候选做 Real-ESRGAN 小样验证；只有前后对比稳定、体积可控时，才考虑人工确认后写回。
- 生成外部素材依赖清单，供换电脑上课前检查。

## 安全门禁

- 不覆盖输入文件。
- 默认不处理视频。
- 默认不做 AI 清晰化。
- AI 清晰化必须先经过照片候选扫描和可视化抽检。
- 候选扫描只读，不替换 PPTX 内媒体。
- 清晰化抽样验证只读，不替换 PPTX 内媒体。
- 清晰化写回必须输出新 PPTX，并重新导出 PDF/PNG 复核后才允许进入 final。
- 文件小于阈值时跳过。
- 压缩后未减少至少 5% 时不替换。
- 处理失败时保留原媒体。
- 所有输出必须生成报告和 manifest。
- 自动重试仅用于已知 PowerPoint/COM 瞬时失败，不改变课件内容。
- 外部链接只报告，不自动删除或改写。

## 回滚点

- 删除 `.media-optimized.pptx` 或 `.final.pptx`。
- 回到对应 `.normalized.pptx`。
- 参考 `media-optimization-report.csv` 中的 `Action` 和 `Reason` 追踪每个媒体文件。
