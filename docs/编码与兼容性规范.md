# 编码与兼容性规范

## 目标

本工具包需要兼容 Windows PowerShell 5.1、PowerPoint COM 自动化和 VBA 宏导入。仓库内存在中文路径、中文字体名、中文视频关键词和中文文档名，因此编码错误会直接导致脚本无法定位文件、配置读取乱码或 VBA 宏提示异常。

## 文件编码

- `tools/*.ps1`：必须保存为 UTF-8 with BOM。Windows PowerShell 5.1 依赖 BOM 判断脚本编码。
- `examples/*.ps1`：必须保存为 UTF-8 with BOM，避免示例命令中的中文路径乱码。
- `config/*.json`：保存为 UTF-8；脚本读取时必须显式使用 `-Encoding UTF8`。
- `docs/*.md`：保存为 UTF-8。
- `vba/*.bas`：保存为 UTF-8；导入 PowerPoint 前如发现中文乱码，优先通过 VBA 编辑器复制粘贴导入。

## 代码要求

- PowerShell 读取 JSON、VBA、Markdown 等包含中文的文本文件时，必须显式指定 `-Encoding UTF8`。
- PowerShell 生成给 Excel 使用的 CSV 时，必须写入 UTF-8 BOM。
- 不要把中文配置值复制成转码后的乱码文本；发现乱码应回到源文件按 UTF-8 重新读取。
- 修改 `config/physics-ppt-style.config.json` 中的字体、字号、关键词时，需要同步检查 `vba/PhysicsPptCommon.bas` 中对应常量。

## 验证

每次修改后运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File tools\Test-ToolkitFiles.ps1
```

该自检会覆盖必需文件、JSON 结构、颜色格式、PowerShell 语法、PowerShell BOM、VBA `Option Explicit` 和最小 PPTX 样例结构。
