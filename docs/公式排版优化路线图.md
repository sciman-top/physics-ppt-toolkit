# 公式排版优化路线图

## 1. 需求边界

本路线图只处理初中物理 PPT 中的公式排版可读性和复核效率，不自动改写教学内容。

自动化允许处理：

- 普通文本框中的短公式候选。
- 低风险独立公式的字体、字号、颜色、对齐和文本框宽度。
- 固定白名单公式的 MathML/OMML 候选片段导出和后续人工确认转换。
- 复杂公式的 SVG/图片渲染兜底方案。
- 公式候选、处理结果、跳过原因和回滚点报告。

自动化默认不处理：

- 图片、视频截图中的公式识别和重写。
- MathType/OLE 对象全自动迁移。
- 组合对象内部公式拆解。
- 嵌在长句中的公式语义转换。
- 任意 LaTeX/UnicodeMath 到 OfficeMath 的无白名单批量转换。
- 未经人工复核的图片公式 OCR 到 OfficeMath 自动替换。

## 2. 技术栈依据

- PowerPoint COM：继续作为现有批处理入口，负责读取形状、应用低风险样式、导出 PDF 和页面图。
- OfficeMath/OMML：作为重要公式的长期目标格式，适合 PowerPoint 内继续编辑；本仓已有课件中可见 `a14:m` + `m:oMath` 结构，可作为后续写入样板。
- MathML：Microsoft 365 可将 Presentation MathML 导入为 Office Math，也可从 Office Math 导出 MathML，因此适合作为 OCR/TeX 与 OMML 写入之间的中间复核格式。
- LaTeX：Microsoft 365 支持将受限 LaTeX 数学表达式转换为 OMML；但 LaTeX 命令覆盖有限，不能作为无审核全自动入口。
- UnicodeMath：作为白名单公式的稳定线性表示，便于人工复核和生成 MathML/OMML 候选片段。
- MathJax/KaTeX SVG：只作为复杂公式或兼容性失败时的展示型降级方案，优先保证课堂播放视觉稳定，保留源码便于重生成。
- 图片公式 OCR：只能作为候选生成链路，必须先裁剪/识别/归一化/人工确认，再进入 MathML/OMML 写入；默认不替换图片。
- 现有报告链路：沿用 CSV、PDF、页面图和复核索引，新增公式分类与处理证据。

## 3. 目标架构

公式处理按“识别 -> 分类 -> 处理策略 -> 验证 -> 回滚”拆成独立步骤。

模块归宿：

- `config/physics-ppt-style.config.json`：公式字体、字号、颜色和默认开关。
- `tools/Normalize-PhysicsPpt.ps1`：公式候选分类、低风险文本公式样式归一、跳过报告。
- `tools/Invoke-PhysicsPptWorkflow.ps1`：沿用 PDF、页面图和复核索引汇总。
- `tools/Export-FormulaWhitelistSuggestions.ps1`：从 `formula-review.csv` 生成白名单晋升建议，只输出复核证据，不自动改配置。
- `tools/Export-FormulaOmmlCandidates.ps1`：把白名单复核项导出为 MathML 和 `a14:m/m:oMath` 候选片段，不改 PPTX。
- `tools/Apply-FormulaOmmlWhitelist.ps1`：在 PPTX 副本中把精确匹配的白名单文本段落替换为 `a14:m/m:oMath`，原 PPTX 不覆盖。
- `tools/FormulaOfficeMathValidator/`：用 Open XML SDK 验证 PPTX 包结构，并统计 `a14:m` 与 `m:oMath`。
- `tools/Export-FormulaImageCandidates.ps1`：从图片候选清单中筛出疑似公式/物理变量标注图，导出原图和联系表，不改 PPTX。
- `tools/Invoke-FormulaImageOcrProbe.ps1` + `tools/formula_image_ocr_probe.py`：用本地 RapidOCR 对图片候选做只读 OCR 探测，输出复核 CSV/JSON/联系表。
- `tools/Export-PptxVisualAudit.ps1`：导出 PPTX 每页 PNG/PDF，扫描形状越界、疑似文本溢出、小字号、空白页和公式文本，生成视觉审查 CSV/JSON/联系表。
- `tools/Apply-PptxVisualAuditFixes.ps1`：读取视觉审查 CSV，只对可确定的形状越界做副本平移修复；不删页、不改文字、不改原文件。
- 后续 `RemoteVisionReviewer`：可选接入 OpenAI Responses API 等联网视觉模型，对导出的页面图做二审；不得替代本地规则门禁，也不得直接改 PPTX。
- 后续 `OfficeMathWriter`：把当前脚本原型收敛为可复用写入模块，并增加更完整的 PowerPoint/WPS 验证。
- 后续 `FormulaRecognition`：只负责图片公式候选裁剪和 OCR 结果归一化，不直接写 PPTX。
- `docs/公式处理说明.md`：面向使用者的公式处理原则。
- `docs/公式排版优化路线图.md`：实施批次、任务清单、验证门禁和回滚点。
- `vba/`：保持离线宏报告能力，后续只做等价低风险增强。

## 4. 分阶段迁移

### P0 公式分类报告增强

目标：不改 PPT 内容，只把 `FormulaCandidate` 拆出更可用的分类。

任务：

- 增加 `FormulaCandidateClass` 报告项。
- 区分低风险独立文本公式、含分式文本公式、角标样文本、希腊字母/单位符号、长句内公式。
- 保持图片、组合对象、OLE 和复杂对象只报告不处理。

验证门禁：

- 自检脚本通过。
- `-ReportOnly` 输出中能看到公式分类。
- 没有新增 PowerPoint 打开修复风险。

回滚点：删除新增分类逻辑，恢复只输出 `FormulaCandidate`。

### P1 低风险文本公式样式归一

目标：只处理普通文本框中的短独立公式，不转换语义。

任务：

- 对低风险独立公式设置数学字体、公式蓝、合适字号、居中。
- 保留原文本内容。
- 文本框只允许已有安全宽度扩展规则处理，不主动移动对象。
- 输出 `FormulaTextStyleNormalized` 或 `FormulaStyleSkipped`。

验证门禁：

- 自检脚本通过。
- 小样本导出 PDF 和页面图。
- 公式未消失、未越界、未被改写。

回滚点：关闭 `rules.formulaTextStyleDefault` 或恢复输入 PPTX。

### P2 白名单公式转换

目标：把固定物理公式从线性文本转换为可编辑 OfficeMath。

任务：

- 建立白名单映射，如 `P=W/t`、`I=U/R`、`R=U/I`、`η=W有/W总`。
- 每条映射记录源文本、目标 UnicodeMath/OMML、页码、形状名。
- P2a 先导出 MathML 和 OMML 候选片段，不改 PPTX。
- P2b 再做 PPTX 副本写入：只替换精确匹配的公式段落，插入 OfficeMath，不覆盖原文件。
- P2c 通过 PowerPoint 打开、导出 PDF、页面 PNG 视觉审查且错误数不高于基线后，才允许扩大范围。

验证门禁：

- `formula-review.csv` 中每个转换项都有白名单命中和目标表达式。
- MathML/OMML XML 片段可被 XML 解析器加载。
- 每个被转换公式有页面渲染图、联系表和结构化审查结果。
- PowerPoint 打开不触发修复。
- Open XML 校验通过，视觉审查错误数不高于转换前基线。

回滚点：只删除本批输出文件，不覆盖原文件。

### P3 复杂公式 SVG 渲染兜底

目标：对不可稳定转成 OfficeMath 的复杂公式提供展示型高质量方案。

任务：

- 调研并固定 MathJax/KaTeX 渲染参数。
- 生成透明 SVG 或 PNG fallback。
- 将公式源码写入报告或对象元数据，便于后续重生成。
- 当前先落地命令行 SVG 渲染器 `tools/Render-FormulaSvg.mjs`，由 MathJax v4 负责 TeX 到 SVG。
- 下一步副本试验使用 `tools/Apply-FormulaSvgWhitelist.ps1`，读取 `formula-review.csv` 并只替换 `ReviewWhitelistConversion` 项。

验证门禁：

- SVG 在目标 PowerPoint 版本中显示正常。
- PDF 导出清晰。
- 低版本 PowerPoint 或 WPS 有 fallback。
- 原 PPTX 不覆盖，原文本框隐藏不删除，失败项保留原状并写报告。

回滚点：保留原公式对象或原文本框，SVG 输出单独生成。

### P4 图片公式识别到可编辑公式

目标：把图片公式从“不可编辑图片”迁移为“可复核源码 + 可编辑 OfficeMath”，但只走候选、归一化和视觉审查门禁。

任务：

- 先用现有图片候选清单筛出高边缘、低色彩、疑似文字/公式图片。
- 为图片公式输出裁剪图、所在页、媒体文件、候选等级和 OCR 待处理状态。
- 当前先用 RapidOCR 做免费本地文字探测，区分公式/变量标注、普通物理图示和人工目检项。
- 真正的图片公式识别可在后续接入 pix2tex/Pix2Text/TexTeller 等开源模型，但必须先评估模型体积、GPU/CPU 成本和误识别率。
- OCR 结果先进入 LaTeX/MathML/UnicodeMath 归一化，不直接写回 PPTX。
- 归一化到白名单或受控公式源后复用 P2 的 MathML/OMML 候选片段和 OfficeMath 写入链路。
- 对无法稳定识别的图片保留原图，最多提供 SVG/PNG 展示兜底。

验证门禁：

- 每个图片公式候选必须有裁剪图和原图引用。
- OCR 结果必须能回显为 MathML/SVG，并在写回前通过结构和视觉审查门禁。
- 不得用 OCR 结果直接覆盖原图。

回滚点：删除 OCR 候选输出和新 PPTX 副本，原 PPTX 与原图片保持不变。

### P5 复核门禁工作流

目标：把不可自动判断的公式集中为可筛选的复核清单，把可自动判断的白名单公式交给视觉审查门禁。

任务：

- 生成公式复核清单和自动门禁结果。
- 按页码、形状名、公式文本、分类、建议动作排序。
- 从复核清单聚合白名单建议，按表达式完整性、频次、文件覆盖数和现有配置覆盖情况分级。
- 对高风险项只给建议，不自动执行。
- 将 `FormulaStyleSkipped`、`FormulaWhitelistCandidate`、`FormulaTextStyleFailed` 纳入重点复核页。
- 对 `ReviewWhitelistConversion` 项记录 OMML 写入、Open XML 校验、视觉审查基线和自动门禁状态。

验证门禁：

- 复核清单能直接定位页面。
- 每个跳过项有原因。
- 每个自动通过项有 `automationGateStatus=Passed` 和可追踪证据。

回滚点：不修改 PPT，仅删除报告。

### P6 视觉审查与自动修复闭环

目标：把“我能看见页面效果”的审查能力落成可重复门禁，而不是一次性人工目检。

任务：

- 用 PowerPoint 官方导出链路生成 PDF 和每页 PNG。
- 对每页 PNG 做本地视觉指标：空白页、内容贴边、黑白/非白占比、联系表。
- 用 PowerPoint COM 读取形状元数据：越界、文本边界疑似溢出、小字号、公式文本。
- 生成自动确认层：按页给出通过、源课件已知问题、需复核、失败，并输出带状态边框的联系表。
- 有基线时只阻断新增错误；源课件已有空白页、贴边等问题进入 `PassedWithKnownIssues`，不误判为本次回归。
- 自动确认层比较页数、错误/警告类型、页面非白/白色/深色占比变化；同页同严重级别问题即使形状名变化，也按基线问题处理，降低误报。
- 对可安全修复的问题只写入副本；对删除空白页、移动对象、缩放对象等改变教学流程或布局的动作，必须作为单独批次和回滚点。
- 自动修复只允许“与画布相交且无需缩放”的越界形状平移；完全离开画布的对象、需要缩放的大图和可能是有意裁切的图片，只报告不改动。
- 可选联网 AI API 只做二审：读取页面图，输出 JSON 审查意见；本地无 API key 时不影响主流程。

验证门禁：

- 每个输出 PPTX 都有 PDF、页面 PNG、视觉审查 CSV、形状审查 CSV、自动确认 CSV/JSON、联系表和 manifest。
- 视觉审查脚本不能修改原文件。
- 自动确认层只能把新增错误阻断，把新增警告列为复核；不能直接改写 PPTX。
- AI API 输出只能进入报告，不能直接写回 PPTX。

回滚点：删除视觉审查输出或删除修复副本，原 PPTX 不变。

## 9. 风险与回滚

关键风险：

- 公式字体变化导致中文角标显示不一致。
- 文本框扩宽后与相邻对象距离变小。
- 个别 PowerPoint 版本对数学字体回退不同。
- 手写 PPTX 中的 OMML 结构不完整，导致 PowerPoint 打开时触发修复。
- 图片公式 OCR 误识别，导致教学公式含义错误。
- 图片候选筛选会把物理图示、坐标尺、表格等纳入候选，不能把 OCR 文本直接当成公式。
- 规则视觉审查会产生保守告警，例如图示贴边、小幅文本边界超出；这些应进入复核表，不应默认自动改版。
- 联网视觉 API 可能误判空间位置或小字，应作为二审建议而不是唯一判断。

控制方式：

- 只处理低风险独立文本公式。
- 默认不改公式内容。
- 报告每个处理结果。
- 输出新 PPTX，不覆盖原文件。
- OMML 写入前先生成片段和复核清单，写入后必须通过 PowerPoint 打开与 PDF 对比。
- 图片公式默认只输出候选和 OCR 待复核结果，不直接替换。
- RapidOCR 当前只做文字探测，不是专用数学公式识别模型；专用公式 OCR 必须另设模型评测门禁。
- 所有视觉修复先保存新 PPTX；默认只做低风险平移，不做缩放、不删除空白页、不把完全画布外对象移回页面。

回滚动作：

- 设置 `rules.formulaTextStyleDefault=false`。
- 删除本次输出目录中的规范化文件。
- 回到 `_backup_originals` 或原输入 PPTX。
