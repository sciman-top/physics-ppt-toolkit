# 公式处理说明

## 1. 推荐公式方案

初中物理 PPT 的重要公式优先使用 Office 自带公式对象，数学字体使用 Cambria Math。

正文和公式之间应保持清晰层级：

- 正文：微软雅黑，32 pt，不加粗。
- 独立公式：Cambria Math，38 pt。
- 核心公式：Cambria Math，40–42 pt。

---

## 2. 为什么不把所有公式全自动转换

PPT 中的公式来源复杂：

- Office 自带公式对象
- 普通文本框
- MathType 对象
- 图片公式
- 视频截图中的公式
- 组合对象内部文本

脚本很难 100% 正确理解公式边界和物理意义。因此默认策略是：

```text
只扫描和报告疑似公式。
只对白名单精确匹配的低风险独立公式走 OMML 写入 + Open XML 结构校验；逐页视觉审查自动门禁作为显式深度复核。
其它公式只进入复核报告，不自动写回。
```

---

## 3. 角标处理原则

常见角标应使用下标，而不是普通字符靠近。

| 文本 | 规范写法 |
|---|---|
| W有 | W 下标 有 |
| W总 | W 下标 总 |
| W额 | W 下标 额 |
| G物 | G 下标 物 |
| G动 | G 下标 动 |
| R1 | R 下标 1 |
| R2 | R 下标 2 |
| U1 | U 下标 1 |
| U2 | U 下标 2 |
| I1 | I 下标 1 |
| I2 | I 下标 2 |
| P1 | P 下标 1 |
| P2 | P 下标 2 |

---

## 4. 分式处理原则

正式展示时，除法优先使用上下分式，不使用斜杠形式。

| 线性写法 | 正式显示 |
|---|---|
| P = W/t | W 在上、t 在下的分式 |
| η = W有/W总 | W有 在上、W总 在下的分式 |
| I = U/R | U 在上、R 在下的分式 |
| R = U/I | U 在上、I 在下的分式 |

---

## 5. 自动化脚本的公式能力

本工具包默认只生成疑似公式报告，不默认转换公式。

疑似公式识别包括：

- 包含 `=` 的短文本框
- 包含 `/` 的公式样文本
- 包含 `η`、`Ω` 等符号
- 包含 `W有`、`W总`、`G物`、`R1`、`U1`、`I1` 等常见角标写法

报告用于人工复核和后续低风险手动处理。

自动化报告会补充公式候选分类：

- `FormulaCandidate`：疑似公式原文。
- `FormulaCandidateClass`：风险等级、公式类型、长度和跳过原因。
- `FormulaTextStyleNormalized`：低风险独立文本公式已做字体、字号、颜色和居中归一。
- `FormulaStyleSkipped`：公式样式处理被跳过，通常是长句、复杂文本或配置关闭。
- `FormulaWhitelistCandidate`：命中固定物理公式白名单，后续可走 OMML 自动门禁。
- `FormulaConversionSkipped`：命中白名单，但 `formulaConversionDefault=false`，因此不做语义转换。

当前低风险样式归一只处理普通文本框中的短独立公式，不改写公式内容，不把公式转换为 OfficeMath。
如需关闭该能力，可在 `config/physics-ppt-style.config.json` 中设置：

```json
"formulaTextStyleDefault": false
```

完整实施路线见 `docs/公式排版优化路线图.md`。

白名单公式配置位于 `formulaWhitelist`。每条规则包含：

- `name`：公式名称。
- `sourcePattern`：匹配去空白后的原始文本。
- `targetUnicodeMath`：建议转换目标。
- `targetTex`：SVG 渲染目标。
- `note`：复核或自动门禁提示。

工作流会额外生成 `10_复核索引/formula-review.csv`，按单条公式汇总原文、分类、白名单命中、转换状态、样式状态和建议动作。

一键流程会生成 `10_复核索引/formula-whitelist-suggestions/formula-whitelist-suggestions.csv`。该清单只用于扩大白名单前的证据整理：它会按出现频率、文件覆盖数、表达式完整性和已配置白名单覆盖情况，把候选分为 `PromoteCandidate`、`ReviewCandidate`、`ReviewSubscriptOnly`、`ExistingWhitelist` 或 `Rejected`。它不会自动修改 `config/physics-ppt-style.config.json`，也不会自动写回 PPTX。

单独生成白名单建议清单时，可使用：

```powershell
powershell -File tools\Export-FormulaWhitelistSuggestions.ps1 `
  -FormulaReviewCsv "reports\pptx_dir_enhanced_visual_v3\10_复核索引\formula-review.csv" `
  -OutputDir "reports\formula_whitelist_suggestions" `
  -MaxSuggestions 120
```

白名单晋升规则：只有来源文本稳定、目标 `targetUnicodeMath` / `targetTex` 可复核、OMML 片段可解析、PPTX 副本 Open XML 校验通过，才应把建议项写入配置；批量扩大白名单前建议再抽样运行 `-FormulaOmmlVisualAudit` 做渲染复核。初中课件默认保留教材常见写法：`W总=Fs` 应转换为可编辑下标 `W_总=Fs`，但不自动改成 `W_总=F·s`。

复杂公式可先渲染为 SVG 做展示型验证：

```powershell
npm install
node tools/Render-FormulaSvg.mjs --tex "P=\frac{W}{t}" --out reports\formula_svg_smoke\power.svg
```

SVG 路线只保证展示清晰，不等同于 PowerPoint 内可编辑公式。可编辑终态应优先使用 OfficeMath/OMML。

对白名单公式先生成可编辑候选片段时，使用：

```powershell
powershell -File tools\Export-FormulaOmmlCandidates.ps1 `
  -FormulaReviewCsv "reports\formula_route_workflow_check\10_复核索引\formula-review.csv" `
  -OutputDir "reports\formula_omml_candidates"
```

该脚本会输出：

- `formula-omml-candidates.csv`：白名单公式、页码、形状、目标表达式和片段路径。
- `fragments/*.mathml.xml`：可供 Microsoft 365 MathML 导入链路复核的 Presentation MathML。
- `fragments/*.omml.xml`：后续 PPTX 写入器可使用的 `a14:m/m:oMath` 候选片段。

该脚本不修改 PPTX，也不隐藏或删除任何形状。

对白名单候选做可编辑 OfficeMath 副本试验时，使用：

```powershell
powershell -File tools\Apply-FormulaOmmlWhitelist.ps1 `
  -InputPath "PPTX\12.4机械效率（王耀强）.pptx" `
  -FormulaReviewCsv "reports\formula_route_workflow_check\10_复核索引\formula-review.csv" `
  -OmmlCandidateCsv "reports\formula_omml_candidates\formula-omml-candidates.csv" `
  -OutputPath "reports\formula_omml_apply_smoke\12.4机械效率（王耀强）.formula-omml.pptx"
```

该脚本只保存新 PPTX；只替换“整段文本精确等于白名单公式”的段落，避免误删同一文本框中的说明文字。插入失败时保持该段不变并写入 `formula-omml-apply-report.csv`。

写入后建议用 Open XML 验证器复核包结构和 OfficeMath 数量：

```powershell
& "$env:USERPROFILE\.dotnet\dotnet.exe" run `
  --project tools\FormulaOfficeMathValidator\FormulaOfficeMathValidator.csproj `
  -c Release -- `
  "reports\formula_omml_apply_smoke\12.4机械效率（王耀强）.formula-omml.pptx" `
  --max-errors 10
```

验证通过不代表视觉最终通过，还必须用 PowerPoint 导出 PDF 和页面 PNG 进入视觉审查门禁；人工目检只用于抽样或异常定位。

本地规则视觉审查可使用：

```powershell
powershell -File tools\Export-PptxVisualAudit.ps1 `
  -InputPath "reports\formula_omml_apply_smoke\12.4机械效率（王耀强）.formula-omml.pptx" `
  -OutputDir "reports\formula_omml_visual_audit" `
  -ContactSheet
```

该脚本输出：

- `pages/slide-xxx.png`：每页渲染图。
- `*.pdf`：固定版式 PDF。
- `pptx-visual-audit.csv`：空白页、贴边、越界、疑似溢出、小字号、公式文本等审查项。
- `pptx-shape-audit.csv`：形状位置、字号、字体、公式文本候选。
- `pptx-visual-audit.contact-sheet.png`：全页联系表，便于直接看图复核。

自动确认门禁可继续读取上述视觉审查产物：

```powershell
powershell -File tools\Export-PptxVisualConfirmation.ps1 `
  -VisualAuditDir "reports\formula_omml_visual_audit" `
  -ContactSheet
```

该脚本输出：

- `pptx-visual-confirmation.csv`：按页给出 `Passed`、`PassedWithKnownIssues`、`NeedsReview` 或 `Failed`。
- `pptx-visual-confirmation-manifest.json`：汇总 `automationGateStatus`、新增错误、需复核页和失败页。
- `pptx-visual-confirmation.contact-sheet.png`：带颜色状态边框的确认联系表。

如果提供 `-BaselineVisualAuditDir`，脚本会把源课件已有问题标记为 `PassedWithKnownIssues`，只把本次新增错误作为失败。视觉审查和自动确认脚本都不修改 PPTX。白名单文本公式的自动转换以视觉确认门禁“无新增错误”为通过条件；联网视觉 API 可作为二审读取这些页面图，但输出只能进入报告，不能直接写回 PPTX。

对视觉审查中明确的形状越界做低风险副本修复时，使用：

```powershell
powershell -File tools\Apply-PptxVisualAuditFixes.ps1 `
  -InputPath "reports\formula_omml_apply_smoke\12.4机械效率（王耀强）.formula-omml.pptx" `
  -VisualAuditCsv "reports\formula_omml_visual_audit\pptx-visual-audit.csv" `
  -OutputPath "reports\formula_omml_visual_fixed\12.4机械效率（王耀强）.formula-omml.visual-fixed.pptx"
```

该脚本目前只处理 `ShapeOutOfSlideBounds`，并且只对仍与画布相交、无需缩放即可回到安全区域的形状做平移修复。完全在画布外的对象、需要缩放才放得下的对象、可能是有意裁切的大图，都会写入跳过原因并保留原状。它不会删除空白页，不会改公式文本，也不会覆盖原 PPTX。

对白名单候选做 PPTX 副本试验时，使用：

```powershell
powershell -File tools\Apply-FormulaSvgWhitelist.ps1 `
  -InputPath "PPTX\12.4机械效率（王耀强）.pptx" `
  -FormulaReviewCsv "reports\formula_route_workflow_check\10_复核索引\formula-review.csv" `
  -OutputPath "reports\formula_svg_apply_smoke\12.4机械效率（王耀强）.formula-svg.pptx"
```

该脚本只保存新 PPTX，原文本框会被隐藏而不是删除；插入失败时保留原公式并写入 `formula-svg-apply-report.csv`。

## 6. 图片公式的处理边界

图片公式可以转换成可编辑 OfficeMath，但不能直接跳过识别、归一化和视觉门禁。推荐链路是：

```text
图片候选定位 -> 公式区域裁剪 -> OCR 得到 LaTeX/MathML -> 归一化到白名单/受控公式源 -> 生成 MathML/OMML 候选 -> 写入 PPTX 副本 -> Open XML + PDF/页面图视觉门禁
```

当前工具包只允许做到候选和片段阶段：

- 图片公式默认不自动 OCR、不自动替换。
- OCR 结果必须先以 LaTeX/MathML/UnicodeMath 源码进入报告；只有归一化到白名单或受控公式源后，才能进入自动写回门禁。
- 无法稳定识别的图片公式保留原图，必要时只做 SVG/PNG 展示兜底。
- 任何可编辑公式写入都必须保存为新 PPTX，并保留原对象作为回滚点。

从已有图片候选清单筛出疑似公式/变量标注图片时，使用：

```powershell
powershell -File tools\Export-FormulaImageCandidates.ps1 `
  -ImageCandidateCsv "reports\final_image_candidates\pptx-image-candidates.csv" `
  -OutputDir "reports\formula_image_candidates" `
  -ExportCandidates `
  -ContactSheet
```

该脚本输出：

- `formula-image-candidates.csv`：候选图片、页码、视觉特征、候选等级和建议动作。
- `formula-image-candidates.contact-sheet.png`：候选联系表，便于快速目检。
- `formula-image-candidates/`：从 PPTX 副本中导出的候选原图。

对候选图做免费本地 OCR 探测时，使用：

```powershell
powershell -File tools\Invoke-FormulaImageOcrProbe.ps1 `
  -FormulaImageCandidateCsv "reports\formula_image_candidates\formula-image-candidates.csv" `
  -OutputDir "reports\formula_image_ocr" `
  -MaxImages 20 `
  -MaxPixels 1500000
```

该脚本使用本地 Python + RapidOCR，只生成复核报告：

- `formula-image-ocr.csv`：OCR 文本、置信度、公式信号、建议动作。
- `formula-image-ocr.contact-sheet.png`：候选图与 OCR 文本联系表。
- `SuggestedAction=ReviewOcrFormulaText`：优先归一化到白名单或受控公式源，再决定是否可转为 MathML/OMML。
- `SuggestedAction=ReviewOcrPhysicsLabels`：多为物理图示变量标注，不能直接当作公式。
- `SuggestedAction=ManualVisualReview`：OCR 不足以判断，保留人工目检。

RapidOCR 当前只是文字 OCR 探测，不是专用数学公式识别模型。若后续需要把图片公式自动转 LaTeX，可评估 pix2tex/LaTeX-OCR 等开源模型，但必须先把模型下载、运行成本、误识别率和视觉审查门禁纳入任务清单。

## 7. 增强一键：白名单公式转可编辑 OMML

默认一键处理仍以安全排版、PDF、复核清单、白名单建议、OMML 副本和 Open XML 结构校验为主，不静默改变公式语义。全量逐页 PNG 视觉审查、视觉修复副本和 OMML 视觉审查属于重型复核环节，默认不启用。需要把白名单命中的独立公式转换成 PowerPoint 可编辑公式时，使用中速增强开关：

```powershell
powershell -File tools\Invoke-PhysicsPptWorkflow.ps1 `
  -InputPath "PPTX" `
  -Recurse `
  -Mode NormalizeAndPdf `
  -SkipPreflightReport `
  -ApplyFormulaOmmlWhitelist `
  -OpenGeneratedPptx
```

也可以直接运行仓库根目录的 `一键规范化导出并转换可编辑公式.cmd`。

增强链路会新增：

- `14_公式OMML副本/`：基于规范化/视觉修复后的 PPTX 生成可编辑 OfficeMath/OMML 副本。
- `14_公式OMML副本/_candidates/`：从 `formula-review.csv` 导出的 MathML/OMML 片段和候选清单。
- `15_公式OMML审查/`：默认包含 Open XML 校验报告；只有显式传入 `-FormulaOmmlVisualAudit` 时才生成 PowerPoint 导出的 PDF/PNG、视觉审查 CSV 和联系表。
- `10_复核索引/formula-whitelist-suggestions/`：公式白名单建议清单，只用于后续人工晋升和自动门禁复测，不自动改配置。

该链路只处理 `formula-review.csv` 中 `SuggestedAction=ReviewWhitelistConversion` 的白名单公式；原始 PPTX、普通规范化 PPTX 和视觉修复副本都不会被覆盖。`review-manifest.json` 中会记录每个 OMML 副本的 `automationGateStatus` 和 `automationGateReason`。

需要深度视觉闭环时，再显式追加：

```powershell
-IncludeReviewArtifacts -IncludeVisualAudit -ApplyVisualAuditFixes -FormulaOmmlVisualAudit
```
